## Deep story analysis — themes

### What went well

- **Clear scope boundaries** in story Dev Notes (especially 1.1, 1.4, 1.5) reduced duplicate work (Docker, integration tests, full E2E journeys deferred appropriately).
- **Architecture and epics as sources of truth** were cited consistently in stories; `project-context.md` stayed aligned (e.g. scaffold status, stack table).
- **Review loops improved quality:** 1.1 dependency-pinning decision (lockfiles authoritative), 1.3 root-only `.env.example` vs per-package, pinned `concurrently`, env loading documentation.
- **1.5 bootstrap** (`npm run bootstrap`, `scripts/bootstrap.mjs`) closed the gap between “documented steps” and repeatable fresh-clone setup; review hardening (File List accuracy, Windows note, copy error handling) was concrete and shippable.

### Struggles and surprises (systems, not blame)

- **Generator ergonomics:** Fastify scaffold initially landed at repo root; relocation into `api/` was required. **Lesson:** run generators with explicit target paths and verify tree before committing.
- **Env file strategy evolved:** Story 1.3 assumed root `.env.example`; 1.5 moved to **per-package** `client/.env.example` and `api/.env.example` with bootstrap copying — good outcome, but it is a **process lesson** to lock env layout earlier or document “may evolve in 1.5” in 1.3 to avoid churn for contributors mid-epic.
- **1.4 still in review** with an open **README** finding (Linux/WSL `--with-deps` guidance). **Lesson:** close documentation polish items in the same review pass as implementation so “review” does not stall epic closure.

### Technical debt and carry-forward

- Open **review finding** on 1.4: README E2E section — add guidance for Linux/WSL contributors (`playwright install --with-deps` or link to Playwright install docs) so local behavior matches CI.
- No production incidents (scaffold-only epic). **Debt** is mostly **documentation and consistency** (env examples, Playwright install parity), not runtime defects.

### Testing and quality

- Unit tests and lint run at root and package level; Playwright smoke is hermetic (`about:blank`), matching Epic 1 intent.
- CI includes a dedicated Playwright job — aligns with architecture direction for a growing pipeline.
