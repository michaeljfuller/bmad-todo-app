## Deep story analysis — themes

### What went well

- **Tests moved with the API:** `api/test/integration/` grew with each story; the epic’s “mandatory in-epic” testing bar was met without a separate testing epic.
- **Single contract thread:** One REST prefix (`/todos`), one error envelope, one DELETE convention (**204**), documented through README, Postman, and OpenAPI — matches `project-context.md` anti-patterns (no mixed prefixes or shapes).
- **Concrete review fixes:** `2-2` added README interim REST docs and **`--test-concurrency=1`** to eliminate **`DATABASE_PATH`** races across parallel test files — a systems-level fix, not a one-off hack.
- **Framework learnings captured in story records:** e.g. `@fastify/autoload` prefix behavior (`routes/todos/index.js` vs colliding `todos.js` + `root.js`); fastify-cli **`--options`** so `app.js` options (Ajv `removeAdditional: false`) apply in tests — these notes shorten the path for Epic 3/Epic 4 contributors.
- **Epic 1 carry-forward closed:** Epic 1 retro noted root CI did not run API tests; **`2-4`** added an **`api-integration`** job, addressing that gap.

### Struggles and surprises (systems, not blame)

- **Scaffold + generators:** Autoload and CLI option merging behaved in ways that are easy to miss until integration tests fail — the team responded by documenting and fixing in **`2-2`/`2-3`** rather than papering over in tests only.
- **Story template drift:** Some “previous story file not present” boilerplate in early Epic 2 drafts was stale once artifacts existed — a **process** nudge to regenerate or trim story intros when files land.
- **OpenAPI ergonomics:** Generated path presentation (e.g. trailing slash nuance) required README honesty — contract is still authoritative, but contributors need the one-liner to avoid false alarms during review.

### Technical debt and carry-forward

- **Low / documented:** Optional future hardening (e.g. richer OpenAPI diff in PRs) is nice-to-have; current **`openapi-contract.test.js`** smoke is aligned with story scope.
- **Epic 4:** **`/health`** and **`/ready`** remain explicitly out of scope for Epic 2 — no debt against this epic’s definition.

### Testing and quality

- Integration coverage: list/create, validation, PATCH/DELETE, 404, persistence across requests, OpenAPI JSON smoke, CORS preflight — strong baseline for Epic 3 E2E against a real API.
